# Avaliação de Desempenho SCI — Skill Completa

## Visão Geral

Esta skill processa qualquer quantidade de avaliações de desempenho semestrais da SCI — de um ou múltiplos colaboradores simultaneamente — e produz dois documentos `.docx` profissionais **por colaborador**. Todos os critérios metodológicos da SCI estão embutidos — não é necessário re-anexar os manuais.

**Outputs gerados por colaborador:**
1. `relatorio_[sobrenome]_[mes][ano].docx` — Análise do colaborador
2. `analise_avaliador_[sobrenome]_[mes][ano].docx` — Análise do processo avaliativo e do avaliador

---

## Passo 0 — Identificação e Agrupamento por Colaborador

**Este passo é obrigatório antes de qualquer análise.**

Leia todos os PDFs recebidos e agrupe-os por colaborador usando o campo **Nome** do cabeçalho de cada avaliação.

```
Para cada PDF lido:
  - Extraia: Nome completo, Cargo, Departamento, Avaliador, Data do ciclo
  - Agrupe todos os ciclos do mesmo colaborador juntos

Resultado esperado:
  Colaborador A → [ciclo1, ciclo2, ciclo3 ...]
  Colaborador B → [ciclo1, ciclo2 ...]
  ...
```

**Regras de agrupamento:**
- Mesmo nome exato → mesmo colaborador (mesmo que cargo tenha mudado entre ciclos)
- Nomes diferentes → colaboradores distintos, processar separadamente
- Em caso de ambiguidade (nome parcialmente diferente), perguntar ao usuário antes de prosseguir

**Antes de iniciar a análise, informe ao usuário:**
> "Identifiquei N colaborador(es): [lista de nomes]. Vou gerar 2 documentos para cada um. Confirma?"

Aguarde confirmação antes de continuar, a menos que haja apenas 1 colaborador.

---

## Passo 1 — Leitura e Extração dos Dados

Para **cada colaborador** identificado no Passo 0, extraia os dados de todos os seus ciclos:

```
- Data do ciclo (ex: Março/2024)
- Período avaliativo (ex: 01/01/2024 a 29/02/2024)
- Nome, cargo, departamento, unidade, avaliador
- IDP (valor numérico + categoria textual)
- Para cada competência avaliada:
  - Nome da competência
  - Para cada comportamento: descrição + nota textual
  - Comentários do avaliador (se houver)
- Observações finais
```

Ordene os ciclos de cada colaborador cronologicamente antes de analisar.

**Processamento sequencial:** processe um colaborador de cada vez, do início ao fim (análise + 2 documentos), antes de iniciar o próximo. Informe o usuário sobre o progresso:
> "Gerando documentos de [Nome] (1/N)..."
> "Gerando documentos de [Nome] (2/N)..."

---

## Passo 2 — Critérios Metodológicos da SCI

Consulte o arquivo `references/criterios-sci.md` para a metodologia completa (régua de intensidade, escala IDP, competências por nível e cargo, premissa de avaliação de líderes). Esses dados são necessários para contextualizar as notas e o IDP corretamente nos documentos.

---

## Passo 3 — Análise do Colaborador

Monte a análise com estas seções obrigatórias:

### 3.1 Perfil Geral
- Trajetória na empresa (tempo, cargos, promoções identificadas)
- Característica dominante do perfil (baseada no padrão das notas ao longo do tempo)
- Principal ativo e principal desafio recorrente

### 3.2 Matriz Evolutiva
Tabela com todas as competências avaliadas × todos os ciclos. Colunas: competência/comportamento, nota por ciclo (abreviada), tendência (↑↓→~). Inclua coluna de tendência calculada comparando primeiro e último ciclo disponível.

### 3.3 Itens com Evolução Positiva
Para cada item que melhorou: o que era, o que virou, em qual ciclo a mudança ocorreu, evidência do comentário do avaliador.

### 3.4 Itens com Queda ou Estagnação
Idem para regressões. Destaque especial para itens cronicamente baixos (mesma nota em 3+ ciclos).

### 3.5 Pontos Positivos Atuais
Baseado no ciclo mais recente. Use caixas visuais destacadas com ícone ✅.

### 3.6 Pontos a Melhorar
Baseado no ciclo mais recente + padrão histórico. Use caixas visuais com ícone ⚠️. Marque como URGENTE/RECORRENTE itens presentes em 3+ ciclos.

### 3.7 Plano de Desenvolvimento
Tabela de 3 colunas: Curto Prazo (6 meses) / Médio Prazo (12 meses) / Longo Prazo (18+ meses).

### 3.8 Consideração Final
Parágrafo síntese conectando o IDP atual com o potencial de evolução e a principal alavanca de desenvolvimento.

---

## Passo 4 — Análise do Avaliador

### Premissa Obrigatória (leia antes de analisar)
Para líderes, o resultado da equipe É evidência válida e esperada para avaliar competências de liderança. Um time com indicadores cronicamente ruins evidencia falha na assertividade das orientações, nas cobranças ou na escalação de problemas. NÃO classifique como erro metodológico quando o avaliador usa resultado do setor para justificar nota de liderança — isso é correto dentro da metodologia SCI.

### 4.1 Pontos Positivos do Avaliador
Analise e documente:
- Continuidade e visão longitudinal
- Registros de evolução com evidências específicas
- Honestidade e transparência nas críticas
- Vinculação correta entre postura do líder e resultado do time
- Visão holística do colaborador além das notas

### 4.2 Problemas Genuínos (apenas estes são válidos)
Procure exclusivamente por:
- **Canal errado para ocorrências operacionais** (ex: usar nota de competência comportamental para registrar batidas de ponto)
- **Incoerência nota × comentário** (o texto justifica uma nota diferente da atribuída)
- **Comentários duplicados** entre competências distintas no mesmo ciclo
- **Ausência de PDI documentado** para comportamentos cronicamente baixos (3+ ciclos)
- **Comentários genéricos** em notas de liderança que não especificam qual comportamento do líder causou o problema (oportunidade de melhoria, não erro)

### 4.3 Recomendações
Para cada problema identificado: recomendação concreta para o avaliador E para o processo/sistema da empresa.

# Critérios Metodológicos — Avaliação de Desempenho SCI

## 1. Calendário

- **Periodicidade:** Semestral
- **Ciclo de Março:** referência setembro do ano anterior a fevereiro do ano corrente
- **Ciclo de Setembro:** referência março a agosto do ano corrente
- **Quem participa:** CLT com mínimo 6 meses de empresa
- **Quem não participa:** CLT com menos de 6 meses, estagiários, cooperados, PJ, menor aprendiz
- **Troca de avaliador:** quem trocou de superior é avaliado pelo que foi responsável por mais tempo no período

---

## 2. IDP — Índice de Desempenho Profissional

Número único que resume o desempenho do colaborador. Varia entre 0,00 e 1,33.

| Faixa IDP | Categoria |
|-----------|-----------|
| 0,00 a 0,66 | Não atende |
| 0,67 a 0,80 | Atende parcialmente |
| 0,81 a 1,00 | Atende |
| 1,01 a 1,10 | Entrega com destaque |
| 1,11 a 1,33 | Alta performance |

**Atenção ao contextualizar:** Um IDP de 0,87 está na faixa "Atende" — não é crítico, mas está no limite inferior dessa faixa. Um IDP de 0,94–0,96 está confortavelmente na faixa "Atende".

---

## 3. Régua de Intensidade (por comportamento)

Cada comportamento individual é avaliado com uma das cinco notas abaixo:

| Nota numérica | Descrição |
|---------------|-----------|
| 0,00 | Não atende |
| 0,33 | Atende minimamente |
| 0,66 | Atende parcialmente |
| 1,00 | Atende as expectativas |
| 1,33 | Supera as expectativas |

**Nas avaliações reais, as notas aparecem como texto:** "Não atende", "Atende minimamente", "Atende parcialmente", "Atende as expectativas", "Supera as expectativas".

**Conversão para análise comparativa:**
- "Supera" = 1,33
- "Atende" = 1,00
- "Parcial" / "Atende parcialmente" = 0,66
- "Mínimo" / "Atende minimamente" = 0,33
- "Não atende" = 0,00

## 5. Comportamentos por Competência (Set/2024 em diante)

### APRENDIZADO CONTÍNUO
1. Aprende rapidamente novos conceitos técnicos e os aplica para resolver problemas práticos.
2. Adapta-se a novas ferramentas, metodologias e orientações, buscando continuamente aprimorar seu trabalho.

### AUTONOMIA
1. Realiza com sucesso suas atividades sem necessidade de constante supervisão.
2. Gerencia seu tempo de forma eficiente, executando todas as suas tarefas.

### COMPROMETIMENTO
1. Exerce suas atividades com dedicação e consistência.
2. Foca no resultado final do trabalho, não apenas ao que compete a si.

### COMUNICAÇÃO
1. Fala e escreve com clareza, respeito e objetivo, de forma que suas mensagens sejam de fácil compreensão.
2. Ouve e lê atentamente, assegurando que compreende completamente as informações antes de responder ou agir.

### GESTÃO DE CONFLITOS *(líderes)*
1. Identifica e resolve conflitos de forma construtiva.
2. Resolve conflitos o mais breve possível, com inteligência emocional.

### LIDERANÇA *(líderes)*
1. Desenvolve a equipe através de feedback e orientação constante, alinhado ao propósito da empresa.
2. Treina, capacita e desenvolve as pessoas nas questões comportamentais.

### LIDERANÇA TÉCNICA *(líderes)*
1. Realiza encontros e reuniões com equipe, pares e superiores para alinhar procedimentos e resolver problemas, registrando em ata.
2. Concede autonomia para a equipe, delegando e dando suporte técnico sempre que necessário.

### ORIENTAÇÃO AO CLIENTE
1. Compreende as necessidades dos clientes (internos e externos) e busca superar suas expectativas.
2. Responde prontamente a demandas e feedbacks dos clientes (internos e externos), buscando soluções rápidas e eficazes.

### RESPONSABILIDADE
1. Entrega as tarefas conforme as prioridades e prazos combinados.
2. Cumpre a jornada de trabalho conforme acordado, respeitando os horários e limites estabelecidos.

### SOLUÇÃO DE PROBLEMAS
1. Utiliza conhecimento técnico para entregar soluções assertivas, com rapidez e seguindo os processos internos.
2. Entrega soluções definitivas, garantindo que o problema seja resolvido por completo.

### TRABALHO EM EQUIPE
1. Relaciona-se com empatia e de maneira positiva com todas as pessoas, em favor de um objetivo comum.
2. Mostra-se disponível para ajudar a equipe e compartilha informações de forma proativa.

---

## 6. Premissa de Avaliação de Líderes

**Regra fundamental:** Para líderes, o resultado da equipe é evidência direta e válida para avaliar competências de liderança.

**Justificativa:** O papel do líder é produzir resultados através das pessoas. Um time com indicadores cronicamente ruins evidencia que:
- As orientações não estão sendo suficientemente assertivas, **ou**
- Os feedbacks corretivos não estão chegando com a firmeza necessária, **ou**
- Problemas que deveriam ser escalados à coordenação não estão sendo escalados.

**Implicação para análise:** Quando um avaliador usa resultado do setor para justificar nota em competências de Liderança, Liderança Técnica ou Orientação ao Cliente — isso é **metodologicamente correto**, não um erro. O que pode ser aprimorado é a especificidade do comentário: detalhar qual comportamento do líder causou ou perpetuou o problema do setor torna o feedback mais acionável.

---

## 7. Classificação de Problemas no Processo Avaliativo

### Problemas genuínos (classificar como erro):
- Usar nota de competência comportamental para registrar ocorrência operacional (ex: ponto, absenteísmo)
- Incoerência direta entre nota numérica e justificativa textual
- Comentários literalmente idênticos em competências distintas no mesmo ciclo
- Comportamento cronicamente baixo (3+ ciclos) sem PDI formal documentado

### Oportunidades de melhoria (não são erros):
- Comentários de liderança que usam resultado do setor sem especificar o comportamento do líder causador
- Ausência de comentário em notas positivas (poderia reforçar o que funcionou)
- Menção a membros da equipe sem indicar ação específica esperada do líder

### NÃO é erro (corrigir análise se classificado assim):
- Usar resultado do setor como evidência para notas de Liderança, Liderança Técnica ou Orientação ao Cliente
- Manter nota baixa em ciclos consecutivos quando o problema persiste
- Mencionar resultado do time ao avaliar competências de gestão de pessoas

Seções obrigatórias (na ordem):

1. **Capa** — título, tabela avaliador/contexto, nota metodológica sobre a análise

   - Incluir caixa com o princípio + revisão de análises anteriores se aplicável
3. **Seção 2** — Visão Geral (tabela 2 colunas: pontos fortes × oportunidades)
4. **Seção 3** — Pontos Positivos com Evidências
   - Tabela de evidências: Ciclo | Competência | Nota | Comentário | Análise
5. **Seção 4** — Oportunidades de Melhoria — Problemas Genuínos
   - Tabela de evidências para cada problema
   - Caixa 💡 com recomendação após cada problema
6. **Seção 5** — Recomendações Consolidadas (para o avaliador + para o processo da empresa)
7. **Seção 6** — Consideração Final
8. **Rodapé** — data de geração, caráter analítico/desenvolvimento

## Regras Críticas docx-js

- **NUNCA** usar `\n` dentro de TextRun — criar Paragraph separado
- **NUNCA** usar unicode bullets (`•`, `\u2022`) — usar `LevelFormat.BULLET` com numbering config
- **NUNCA** usar `WidthType.PERCENTAGE` em tabelas — sempre `WidthType.DXA`
- **SEMPRE** definir `columnWidths` na tabela E `width` em cada célula (devem somar igual)
- **SEMPRE** usar `ShadingType.CLEAR` (nunca `SOLID`) para fundo de células
- PageBreak deve estar dentro de um Paragraph, nunca sozinho
- Para US Letter: 12240 × 15840 DXA | Para A4: 11906 × 16838 DXA

## Template JavaScript Completo

```javascript
const {
  Document, Packer, Paragraph, TextRun, Header, Footer,
  ImageRun, AlignmentType, HeadingLevel, LevelFormat,
  Table, TableRow, TableCell, BorderStyle, WidthType,
  ShadingType, VerticalAlign, PageNumber, PageBreak
} = require('docx');
const fs = require('fs');
const path = require('path');

// ─── Carrega imagens ────────────────────────────────────────────────────────
const ASSETS = '/mnt/skills/user/docx-padrao-sci-sistemas/assets';
const imgHeaderLogo   = fs.readFileSync(path.join(ASSETS, 'header_logo.png'));
const imgHeaderBanner = fs.readFileSync(path.join(ASSETS, 'header_banner.png'));
const imgFooterBanner = fs.readFileSync(path.join(ASSETS, 'footer_banner.png'));

// ─── Helpers ────────────────────────────────────────────────────────────────
const exoRun = (text, opts = {}) => new TextRun({
  text,
  font: 'Exo 2.0',
  size: opts.size ?? 24,
  bold: opts.bold ?? false,
  color: opts.color,
  italics: opts.italics ?? false,
});

const exoParagraph = (text, opts = {}) => new Paragraph({
  alignment: opts.alignment ?? AlignmentType.JUSTIFIED,
  spacing: { after: opts.spacingAfter ?? 120, line: 264, lineRule: 'auto' },
  children: [exoRun(text, opts)],
});

// ─── Cabeçalho ──────────────────────────────────────────────────────────────
// IMPORTANTE: As imagens usam posicionamento absoluto (anchor) atrás do texto.
// A margem superior (top: 2835) garante que nenhum parágrafo do corpo alcance
// a área do cabeçalho. NÃO REDUZA a margem top.
//
// CRÍTICO: Use os valores de transformation EXATAMENTE como abaixo.
// Os banners têm ~21 cm de largura e devem ocupar a página inteira.
const sciHeader = new Header({
  children: [
    new Paragraph({
      children: [
        // Logo SCI — 4139565 × 1173480 EMU = ~11,5 cm × 3,26 cm
        new ImageRun({
          data: imgHeaderLogo,
          type: 'png',
          transformation: { width: 435, height: 123 },
          floating: {
            horizontalPosition: { relative: 'margin', offset: 998855 },
            verticalPosition:   { relative: 'margin', offset: 3371850 },
            behindDocument: true,
            wrap: { type: 'none' },
            allowOverlap: true,
            layoutInCell: true,
          },
        }),
        // Banner principal — 7590790 × 1085850 EMU = ~21 cm (largura total da página)
        new ImageRun({
          data: imgHeaderBanner,
          type: 'png',
          transformation: { width: 797, height: 114 },
          floating: {
            horizontalPosition: { relative: 'column', offset: -678815 },
            verticalPosition:   { relative: 'paragraph', offset: -446405 },
            behindDocument: true,
            wrap: { type: 'none' },
            allowOverlap: true,
            layoutInCell: true,
          },
        }),
      ],
    }),
  ],
});

// ─── Rodapé ─────────────────────────────────────────────────────────────────
// Banner — 7571105 × 728980 EMU = ~21 cm (largura total da página)
const sciFooter = new Footer({
  children: [
    new Paragraph({
      children: [
        new ImageRun({
          data: imgFooterBanner,
          type: 'png',
          transformation: { width: 795, height: 77 },
          floating: {
            horizontalPosition: { relative: 'column', offset: -685800 },
            verticalPosition:   { relative: 'paragraph', offset: 27940 },
            behindDocument: true,
            wrap: { type: 'none' },
            allowOverlap: true,
            layoutInCell: true,
          },
        }),
      ],
    }),
  ],
});

// ─── Documento ──────────────────────────────────────────────────────────────
const doc = new Document({
  styles: {
    default: {
      document: {
        run: { font: 'Exo 2.0', size: 24, language: { value: 'pt-BR' } },
      },
    },
    paragraphStyles: [
      {
        id: 'Heading1', name: 'Heading 1', basedOn: 'Normal', next: 'Normal', quickFormat: true,
        run: { font: 'Exo 2.0', size: 40, bold: true },
        paragraph: { spacing: { before: 0, after: 120 }, outlineLevel: 0 },
      },
      {
        id: 'Heading2', name: 'Heading 2', basedOn: 'Normal', next: 'Normal', quickFormat: true,
        run: { font: 'Exo 2.0', size: 28, bold: true },
        paragraph: { spacing: { before: 240, after: 120 }, outlineLevel: 1 },
      },
    ],
  },

  sections: [{
    properties: {
      page: {
        size:   { width: 11906, height: 16838 },   // A4
        margin: {
          top:    2835,   // ~5 cm — NÃO ALTERE: protege o cabeçalho visual
          bottom: 1559,
          left:   1134,
          right:  1134,
          header: 720,
          footer: 720,
          gutter: 0,
        },
      },
    },
    headers: { default: sciHeader },
    footers: { default: sciFooter },

    children: [
      // ── Título ──────────────────────────────────────────────────────────
      new Paragraph({
        heading: HeadingLevel.HEADING_1,
        alignment: AlignmentType.LEFT,
        spacing: { after: 120, line: 264, lineRule: 'auto' },
        children: [exoRun('TÍTULO', { size: 40, bold: true })],
      }),

      new Paragraph({ children: [exoRun('')] }),

      // ── Subtítulo ────────────────────────────────────────────────────────
      new Paragraph({
        heading: HeadingLevel.HEADING_2,
        alignment: AlignmentType.LEFT,
        spacing: { after: 120, line: 264, lineRule: 'auto' },
        children: [exoRun('Subtítulo', { size: 28, bold: true })],
      }),

      // ── Parágrafo de corpo ───────────────────────────────────────────────
      exoParagraph('Texto do documento aqui.', { spacingAfter: 200 }),
    ],
  }],
});

// ─── Salva ───────────────────────────────────────────────────────────────────
Packer.toBuffer(doc).then(buffer => {
  fs.writeFileSync('/mnt/user-data/outputs/documento_sci.docx', buffer);
  console.log('Documento gerado: documento_sci.docx');
});
```
